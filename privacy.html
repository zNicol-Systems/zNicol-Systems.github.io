Resumen rápido

zNicol System's es un bot de Discord diseñado para ayudar a comunidades con herramientas de moderación, seguridad, registros de auditoría, protección anti-links, protección anti-scam, comandos administrativos y funciones de configuración para servidores.

🔐 Seguridad

Ayuda a detectar enlaces sospechosos, spam, invitaciones no deseadas y posibles intentos de scam.

📋 Logs

Puede registrar acciones importantes del servidor, como bans, cambios de canales, roles o mensajes.

⚙️ Configuración

Permite configurar funciones mediante comandos dentro de Discord y posibles paneles externos.

Este bot no vende información personal, no comparte datos con anunciantes y no usa la información para crear perfiles comerciales de los usuarios.

1. Política de Privacidad

Esta Política de Privacidad explica cómo zNicol System's puede recopilar, procesar, almacenar o utilizar cierta información necesaria para el funcionamiento del bot dentro de servidores de Discord.

Al invitar, configurar o utilizar el bot en un servidor de Discord, aceptas que el bot procese información limitada y necesaria para cumplir sus funciones.

Esta política aplica únicamente al uso de zNicol System's. No controla ni reemplaza las políticas propias de Discord Inc. ni las normas internas de cada servidor donde el bot sea utilizado.

El objetivo principal del tratamiento de datos es permitir funciones como:

  • Moderación de servidores.
  • Protección contra spam, scams y enlaces no permitidos.
  • Registro de acciones administrativas.
  • Configuración de canales, roles y comandos.
  • Mejora de seguridad dentro de comunidades.
  • Correcto funcionamiento de comandos slash y comandos con prefijo.

2. Responsable del bot

El responsable del funcionamiento de esta aplicación es el equipo o propietario de zNico Community, encargado del desarrollo, mantenimiento y administración de zNicol System's.

La administración de cada servidor también puede decidir cómo configurar el bot, qué canales usar para logs, qué roles tendrán permisos y qué sistemas estarán activos.

Si eres usuario de un servidor donde está el bot y tienes dudas sobre una acción de moderación, primero debes contactar con los administradores de ese servidor.

3. Datos que puede procesar el bot

Para funcionar correctamente, el bot puede procesar datos básicos disponibles dentro de Discord. Estos datos se utilizan únicamente para ejecutar funciones del bot.

3.1 Datos de servidores

  • ID del servidor.
  • Nombre del servidor.
  • ID de canales configurados.
  • ID de roles configurados.
  • Configuraciones de módulos activados o desactivados.
  • Canal de logs configurado por los administradores.
  • Prefijo o ajustes personalizados del servidor.

3.2 Datos de usuarios

  • ID de usuario de Discord.
  • Nombre de usuario o tag visible en Discord.
  • Avatar público del usuario, si Discord lo proporciona.
  • Roles dentro del servidor.
  • Información necesaria para eventos de moderación.
  • Estado de miembro dentro de un servidor, cuando sea necesario para logs o seguridad.

3.3 Datos de mensajes

  • Contenido de mensajes cuando sea necesario para sistemas anti-links, anti-spam o comandos.
  • ID del mensaje.
  • ID del canal donde se envió el mensaje.
  • ID del autor del mensaje.
  • Fecha aproximada del evento.
  • Información necesaria para detectar enlaces prohibidos o sospechosos.

3.4 Datos de moderación

  • Acciones como bans, unbans, timeouts o sanciones.
  • Usuario afectado por una acción.
  • Moderador que ejecutó una acción, cuando Discord lo permita mediante audit logs.
  • Razón de la acción, si fue proporcionada.
  • Fecha del evento.
El bot no solicita contraseñas, tokens personales, códigos de autenticación, información bancaria ni datos privados externos a Discord.

4. Cómo se usan los datos

Los datos procesados por el bot se utilizan para brindar funciones de seguridad, administración y moderación dentro de Discord.

Algunos ejemplos de uso son:

  • Detectar si un mensaje contiene enlaces no permitidos.
  • Eliminar mensajes que incumplan la configuración anti-link del servidor.
  • Registrar en un canal de logs acciones importantes del servidor.
  • Identificar al usuario que ejecutó una acción administrativa, cuando sea posible.
  • Aplicar configuraciones guardadas para cada servidor.
  • Responder comandos ejecutados por usuarios autorizados.
  • Evitar abuso, spam o comportamiento sospechoso.
  • Mostrar información de ayuda o configuración del bot.

La información no se utiliza para publicidad, venta de datos ni seguimiento comercial.

5. Sistema de logs y moderación

zNicol System's puede enviar registros a un canal configurado por los administradores del servidor. Estos registros ayudan a mantener transparencia y control sobre acciones relevantes.

Los logs pueden incluir eventos como:

  • Creación, edición o eliminación de canales.
  • Creación, edición o eliminación de roles.
  • Baneos y desbaneos.
  • Mensajes eliminados o editados.
  • Cambios en miembros, roles o apodos.
  • Eventos relacionados con protección anti-link o anti-scam.
  • Acciones de aislamiento o timeout.

La disponibilidad de algunos datos depende de los permisos del bot y de la información que Discord entregue mediante sus eventos y registros de auditoría.

Si el bot no tiene permisos suficientes, algunos registros pueden no enviarse correctamente o pueden mostrarse con información limitada.

6. Uso de Privileged Gateway Intents

Para que algunas funciones trabajen correctamente, el bot puede requerir ciertos Privileged Gateway Intents de Discord.

6.1 Server Members Intent

Este intent permite al bot recibir eventos relacionados con miembros del servidor. Se utiliza para funciones como:

  • Detectar entrada o salida de miembros.
  • Registrar cambios de roles.
  • Registrar cambios de apodo.
  • Gestionar funciones de moderación relacionadas con miembros.

6.2 Message Content Intent

Este intent permite analizar el contenido de mensajes cuando sea necesario. Se utiliza para:

  • Detectar enlaces no permitidos.
  • Detectar posibles scams.
  • Procesar comandos con prefijo.
  • Aplicar reglas anti-spam o anti-link configuradas.

6.3 Presence Intent

Este intent puede utilizarse para funciones relacionadas con estados de usuario si en algún momento son implementadas. Si no es necesario para una función activa, puede permanecer sin uso o deshabilitado según la configuración del bot.

Los intents se usan únicamente para funciones necesarias del bot dentro de Discord, no para vigilancia externa ni seguimiento comercial.

7. Conservación de datos

Los datos pueden conservarse mientras sean necesarios para el funcionamiento del bot o mientras el bot permanezca configurado en un servidor.

Ejemplos de datos que pueden mantenerse:

  • Configuración del servidor.
  • Canal de logs.
  • Roles exentos de ciertos sistemas.
  • Preferencias del sistema anti-link.
  • Registros necesarios para funciones de seguridad o moderación.

Cuando el bot es removido de un servidor, algunos datos de configuración pueden ser eliminados, desactivados o conservarse temporalmente con fines técnicos, de recuperación o prevención de abuso.

8. Seguridad de la información

Se procura proteger la información procesada por el bot mediante prácticas razonables de seguridad, como limitar el acceso a configuraciones, mantener tokens privados y evitar exposición innecesaria de datos.

Sin embargo, ningún sistema conectado a internet puede garantizar seguridad absoluta. Por ello, los administradores de servidores también deben configurar correctamente permisos, roles y canales.

  • No compartas tokens del bot.
  • No otorgues permisos innecesarios.
  • Usa canales privados para logs sensibles.
  • Revisa periódicamente los permisos del bot.
  • Configura correctamente roles exentos.

9. Derechos de usuarios y servidores

Los usuarios y administradores pueden solicitar información sobre el uso de datos relacionados con el bot, cuando sea técnicamente posible y razonable.

Los administradores de servidores pueden:

  • Desactivar módulos del bot.
  • Cambiar canales configurados.
  • Remover el bot del servidor.
  • Solicitar eliminación de configuraciones relacionadas con su servidor.

Los usuarios pueden contactar al equipo responsable si consideran que existe un problema relacionado con privacidad o uso indebido de información.

10. Servicios de terceros

El bot funciona dentro de Discord y depende de la API de Discord para recibir eventos, responder comandos y ejecutar funciones.

También puede usar servicios técnicos para alojamiento, base de datos o despliegue. Estos servicios se utilizan únicamente para mantener el funcionamiento del bot.

  • Discord API para eventos, usuarios, servidores y comandos.
  • Servicios de hosting para ejecutar el bot.
  • Bases de datos para guardar configuraciones necesarias.
  • Servicios web si existe dashboard o página informativa.

El uso de Discord también está sujeto a las propias políticas de Discord.

11. Menores de edad

El bot está diseñado para utilizarse dentro de Discord y no está dirigido específicamente a recopilar información de menores de edad.

El uso de Discord por parte de menores debe cumplir las normas y requisitos de edad establecidos por Discord y por las leyes aplicables en cada país.

12. Cambios en esta política

Esta política puede actualizarse cuando se agreguen nuevas funciones, cambien sistemas internos o sea necesario mejorar la claridad del documento.

La versión actualizada estará disponible en esta misma página. Se recomienda revisar este documento periódicamente.

13. Términos de Servicio

Al invitar o utilizar zNicol System's, aceptas estos Términos de Servicio. Si no estás de acuerdo, debes remover el bot de tu servidor o dejar de utilizar sus funciones.

13.1 Uso permitido

Puedes utilizar el bot para administración, seguridad, moderación y gestión de servidores de Discord, siempre respetando las normas de Discord y las leyes aplicables.

  • Usar comandos de configuración.
  • Activar sistemas de seguridad.
  • Configurar canales de logs.
  • Aplicar moderación de forma responsable.
  • Utilizar el bot en comunidades legítimas.

13.2 Uso prohibido

No está permitido utilizar el bot para:

  • Acosar, atacar o perjudicar usuarios.
  • Violar los Términos de Servicio de Discord.
  • Automatizar spam, raid o abuso.
  • Intentar obtener datos privados no autorizados.
  • Modificar, clonar o explotar el bot sin permiso.
  • Usar el bot para actividades maliciosas.

13.3 Responsabilidad del servidor

Los administradores de cada servidor son responsables de configurar correctamente el bot, sus permisos, canales, roles y sistemas activos.

El mal uso del bot por parte de administradores o moderadores de un servidor no representa necesariamente una acción directa del propietario del bot.

13.4 Disponibilidad

El bot puede estar temporalmente fuera de servicio por mantenimiento, errores técnicos, problemas de hosting, cambios en Discord o actualizaciones.

No se garantiza disponibilidad permanente o libre de errores.

13.5 Cambios en funciones

Las funciones del bot pueden cambiar, agregarse o eliminarse con el tiempo. Algunas funciones pueden requerir permisos adicionales o configuración específica.

13.6 Suspensión de uso

El acceso al bot puede limitarse o removerse de servidores que hagan mal uso del sistema, intenten abusar de sus funciones o incumplan estos términos.

14. Contacto

Para consultas sobre privacidad, términos, eliminación de datos o reportes relacionados con el bot, puedes contactar al propietario o equipo responsable de zNico Community.
